Introduction

Comprendre PurpleForge

Quel problème vient-il résoudre ?

Dans une équipe SOC, la question qui revient le plus souvent après un exercice red team est simple — et redoutable :

« On sait ce que la red team a fait. Mais est-ce qu'on aurait vu quelque chose si c'était un vrai attaquant ? »

Sans outil dédié, la réponse à cette question se perd dans des fichiers Excel éparpillés, des notes Notepad, des emails et des souvenirs. PurpleForge existe pour y répondre clairement, à tout moment.

❌ Sans PurpleForge

  • Rapport red team en PDF — lu une fois, rangé dans un tiroir
  • Aucun suivi des techniques non détectées
  • La blue team ne sait pas quoi construire en priorité
  • Le RSSI ne peut pas mesurer le progrès
  • Chaque exercice repart de zéro
  • Les règles Sigma restent à trouver manuellement

✅ Avec PurpleForge

  • Chaque technique jouée est enregistrée avec son statut
  • Backlog de détection priorisé automatiquement
  • Règles Sigma suggérées en un clic par technique
  • Couverture mesurée en pourcentage, par tactique
  • Historique de tous les exercices comparables
  • Export ATT&CK Navigator pour la direction

Le purple teaming expliqué simplement

Le purple teaming est une pratique collaborative qui fait travailler ensemble la red team (attaquants) et la blue team (défenseurs) — au lieu de les faire s'affronter.

ÉquipeRôleCe qu'elle fait dans PurpleForge
🔴 Red teamSimule l'attaquantDocumente les techniques jouées, les outils utilisés, le résultat
🔵 Blue teamDéfend et détecteQualifie chaque technique, note les règles déployées
🟣 Purple teamCoordonneSuit la couverture globale, pilote la remédiation

MITRE ATT&CK en 5 minutes

MITRE ATT&CK est un catalogue mondial de techniques d'attaque, organisé en 14 tactiques (les étapes d'une attaque) contenant chacune des dizaines de techniques.

🗂️

Les 14 tactiques ATT&CK Enterprise

Reconnaissance · Resource Development · Initial Access · Execution · Persistence · Privilege Escalation · Defense Evasion · Credential Access · Discovery · Lateral Movement · Collection · Command & Control · Exfiltration · Impact

Chaque technique a un identifiant unique (ex. T1003.001 pour LSASS Memory Dump). PurpleForge utilise ce référentiel comme langage commun entre la red et la blue team.

Les règles Sigma

Sigma est un format standard ouvert pour écrire des règles de détection. Une règle Sigma décrit quoi chercher dans les logs pour détecter une technique d'attaque. Elle peut ensuite être convertie pour n'importe quel SIEM (Splunk, Wazuh, Elastic…).

💡

Bonne nouvelle

PurpleForge télécharge automatiquement plus de 2 900 règles Sigma depuis le dépôt communautaire SigmaHQ. Pour chaque technique ATT&CK jouée, il vous propose les règles correspondantes — prêtes à copier dans votre SIEM.

Option A — Installation locale (Windows)

1

Télécharger Python 3.12

Rendez-vous sur python.org/downloads et installez Python 3.12+. Cochez impérativement « Add Python to PATH » lors de l'installation.

2

Cloner le dépôt

PS> git clone https://github.com/Yeni-raamia/purpleforge-compagnon.git
PS> cd purpleforge-compagnon
3

Créer l'environnement virtuel

PS> python -m venv .venv
PS> .\.venv\Scripts\Activate.ps1

⚠️ Erreur « scripts désactivés » ?

Exécutez d'abord : Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

4

Installer les dépendances

(.venv) PS> pip install -r requirements.txt
5

Lancer le serveur

(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

Ouvrez http://localhost:8080 dans votre navigateur.

ℹ️ Premier démarrage

L'application télécharge automatiquement les données ATT&CK (~40 Mo) et les règles Sigma (~30 Mo). Cette opération prend 1 à 3 minutes et n'a lieu qu'une seule fois. Les démarrages suivants sont instantanés.

6

Connexion initiale

Identifiants par défaut : admin / admin. Changez le mot de passe immédiatement via le menu Compte.

Option B — Installation macOS / Linux

# Cloner et entrer dans le dossier
$ git clone https://github.com/Yeni-raamia/purpleforge-compagnon.git && cd purpleforge-compagnon

# Créer et activer l'environnement virtuel
$ python3 -m venv .venv && source .venv/bin/activate

# Installer et lancer
(.venv) $ pip install -r requirements.txt
(.venv) $ uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

Option C — Docker (recommandé en production)

Docker isole l'application et simplifie les mises à jour. C'est la méthode recommandée si vous déployez PurpleForge sur un serveur partagé.

1

Installer Docker Desktop

Téléchargez depuis docker.com/products/docker-desktop et lancez l'application.

2

Lancer en une commande

$ docker compose up

PurpleForge est accessible sur http://localhost:8080. La base de données et les données ATT&CK survivent aux redémarrages grâce aux volumes Docker.

3

Arrêter / Mettre à jour

$ docker compose down          # arrêter
$ git pull && docker compose up --build  # mettre à jour

Vérification de l'installation

# Lancer les tests automatiques (64 tests)
$ python -m pytest tests/ -v
64 passed in 0.75s ✓
🎉

Installation réussie !

Si vous voyez « 64 passed », tout est en ordre. PurpleForge est prêt à l'emploi.

3.1 — Créer votre première campagne

1

Ouvrir la page Campagnes

Cliquez sur Campagnes dans la navigation. La page « Tactical Ops Center » s'ouvre — fond sombre, statistiques globales en haut.

2

Cliquer sur « + Nouvelle Campagne »

Le bouton circulaire en haut à droite ouvre un panneau latéral (drawer).

3

Remplir le formulaire

  • Nom : ex. « Red Team Q3 2026 — Mouvement Latéral »
  • Description : contexte de l'exercice, périmètre testé
  • Tags : ex. APT29, Ransomware, Interne
4

Cliquer « Créer »

La campagne apparaît dans la grille. Cliquez dessus pour accéder à sa page de détail.

🚀

Astuce : Templates APT

Plutôt que de partir de zéro, utilisez les templates préconfigurés : Templates APT → choisissez le groupe (APT28, APT29, Lazarus, FIN7…). La campagne est créée avec les techniques caractéristiques du groupe déjà chargées.

3.2 — Ajouter des techniques depuis la matrice

1

Ouvrir la matrice

Depuis la page de la campagne, cliquez Ouvrir la matrice ATT&CK. La matrice complète s'affiche : 14 tactiques en colonnes, techniques en lignes.

2

Naviguer par tactique

Cliquez sur un onglet de tactique (ex. « Credential Access ») pour voir ses techniques. Chaque technique affiche son identifiant ATT&CK et son nom.

3

Ajouter une technique

Cliquez sur + Ajouter sous la technique jouée pendant l'exercice. Elle apparaît immédiatement dans la campagne (HTMX, sans rechargement de page).

3.3 — Qualifier une détection

C'est le cœur de PurpleForge. Pour chaque technique, la blue team doit répondre : « est-ce qu'on l'aurait détectée ? »

StatutSignificationAction suivante
✗ Non détectéeAucune alerte levée lors de l'exerciceCréer une règle de détection (priorité haute)
⚙ À construireUne règle existe mais n'est pas encore déployéeDéployer et valider en SIEM
✓ DétectéeAlerte levée et validéeDocumenter et passer à la suivante
1

Ouvrir la fiche technique

Sur la page de détail de la campagne, cliquez sur une technique pour déplier sa fiche.

2

Changer le statut

Sélectionnez le statut dans le menu déroulant. La mise à jour est instantanée (HTMX) — la barre de couverture en haut se recalcule immédiatement.

3

Ajouter les notes

  • Note Red Team : outil utilisé, payload, vecteur d'accès, résultat de l'attaque
  • Note Blue Team : règle déployée, ticket SIEM, commentaire technique (Markdown supporté)

3.4 — Les règles Sigma suggérées

1

Cliquer « ⚡ Règles Sigma »

Sur la fiche d'une technique, ce bouton jaune affiche les règles Sigma correspondantes depuis SigmaHQ.

2

Lire la suggestion

Chaque règle indique : titre, logs requis (ex. « Sysmon Event ID 10 »), et le contenu YAML complet.

3

Copier dans votre SIEM

Copiez le YAML et importez-le dans Wazuh, Elastic, Splunk ou tout autre SIEM compatible Sigma. Notez la référence dans la note Blue Team.

Cas pratiques

Tutoriels pas à pas

Cas 1 — Premier exercice post-compromission

📋 Contexte

Votre red team vient de réaliser un exercice de 2 jours. Elle a simulé un attaquant ayant déjà accès à un poste Windows. Techniques jouées : extraction de credentials (Mimikatz), mouvement latéral (PsExec), persistence (tâche planifiée).

1

Créer la campagne

Campagnes → Nouvelle Campagne → Nom : « Exercice Post-Compromission — Juillet 2026 », Tags : interne, Windows

2

Ajouter les techniques depuis la matrice

Matrice ATT&CK → Ajouter :

  • T1003.001 — LSASS Memory Dump (Credential Access)
  • T1021.002 — SMB/Windows Admin Shares (Lateral Movement)
  • T1053.005 — Scheduled Task/Job (Persistence)
3

Qualifier chaque technique

  • T1003.001 → Non détectée — aucune alerte Wazuh sur l'accès LSASS
  • T1021.002 → À construire — règle Sigma identifiée, pas encore déployée
  • T1053.005 → Détectée — alerte levée via Sysmon Event ID 11
4

Récupérer les règles Sigma pour T1003.001

Cliquez ⚡ Règles Sigma → PurpleForge propose « Credential Dumping via Procdump » et « LSASS Access via Task Manager ». Copiez le YAML et créez la règle dans Wazuh.

5

Créer les tâches de remédiation

Pour T1003.001 et T1021.002 → Éditer → assigner à yeni, deadline dans 2 semaines → le board remédiation se met à jour.

6

Exporter pour la direction

⬇ Exporter Navigator → le JSON s'ouvre sur attack-navigator → vue matricielle avec vos 3 techniques colorées. Parfait pour le compte-rendu.

Cas 2 — Utiliser un template APT28

📋 Contexte

Vous souhaitez évaluer votre couverture face aux techniques caractéristiques d'APT28 (Fancy Bear), groupe lié à la GRU russe, connu pour cibler des institutions gouvernementales.

1

Charger le template

Campagnes → Templates APT → choisir « APT28 / Fancy Bear » → Créer depuis ce template. Une campagne est créée avec ~15 techniques caractéristiques du groupe.

2

Parcourir les techniques

La campagne contient des techniques de spearphishing, vol de credentials, mouvement latéral… Chacune est à qualifier selon vos capacités de détection actuelles.

3

Évaluation honnête sans exercice réel

Même sans exercise red team, vous pouvez qualifier les techniques selon votre connaissance de votre SIEM : « est-ce que j'ai une règle pour ça ? » C'est une évaluation de posture utile.

4

Page de couverture

Couverture → la barre empilée montre votre taux de détection face à APT28 par tactique. Si Credential Access est à 0% — c'est une priorité immédiate.

Cas 3 — Piloter la remédiation sur 3 mois

📋 Contexte

Après plusieurs exercices, vous avez un backlog de 20 techniques non détectées. Vous devez organiser le travail de la blue team sur le trimestre.

1

Tableau de bord global → « Points chauds »

Le dashboard identifie les techniques qui reviennent sur plusieurs campagnes. T1055 Process Injection sur 3 campagnes = priorité absolue.

2

Assigner et planifier

Remédiation globale → pour chaque technique : assigner un responsable, fixer une deadline. Le board kanban visualise En cours / Bloqué / Terminé.

3

Suivi hebdomadaire

Statistiques → « Évolution du taux de détection » montre la progression mois par mois. Partagez le graphe lors des réunions de suivi.

4

Relancer un exercice de validation

Après déploiement des règles, créez une nouvelle campagne sur les mêmes techniques → les techniques anciennement Non Détectées devraient passer en Détectées. C'est la preuve concrète du progrès.

Bonnes pratiques & Glossaire

Bonnes pratiques

🏷️ Nommer ses campagnes de façon cohérente

Adoptez un format standard dès le début :

[Type] [Année-Trimestre] — [Thème]
Exemple : Red Team Q3-2026 — Ransomware LockBit
Exemple : Purple Q4-2026 — Accès Initial

📅 Cadence recommandée

FréquenceAction
Après chaque exerciceCréer la campagne, ajouter et qualifier toutes les techniques dans les 48h
HebdomadaireMettre à jour les statuts remédiation, vérifier les deadlines
MensuelRevoir le dashboard global, ajuster les priorités
TrimestrielComparer deux campagnes (fonction Comparer), présenter l'évolution

🎯 Exploiter la vue Comparer

La fonction Comparer (Campagnes → Comparer) affiche côte à côte deux campagnes et met en évidence les techniques qui ont changé de statut. C'est votre indicateur de progrès le plus puissant.

📊 Partager avec la direction

Le fichier ATT&CK Navigator JSON est votre meilleur allié. Importé sur attack-navigator.mitre.org, il produit une heatmap colorée (rouge = non détecté, vert = détecté) compréhensible par n'importe qui, sans jargon technique.

💡

Conseil pratique

Exportez Navigator avant ET après chaque phase de remédiation. La différence visuelle entre les deux matrices est le meilleur argument pour montrer la valeur de votre équipe SOC.
