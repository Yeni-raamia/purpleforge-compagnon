Lancer l'application

Voici les commandes exactes à taper selon ton environnement. À utiliser à chaque fois que tu veux lancer PurpleForge.

🪟 Windows (PowerShell)

Windows 
# 1. Aller dans le dossier projet
PS> cd C:\chemin\vers\purpleforge

# 2. Activer l'env virtuel
PS> .\.venv\Scripts\Activate.ps1

# 3. Lancer le serveur
(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

Ouvre ensuite http://localhost:8080

🍎 macOS / 🐧 Linux (Terminal)

macOS Linux 
# 1. Aller dans le dossier projet
$ cd ~/purpleforge

# 2. Activer l'env virtuel
$ source .venv/bin/activate

# 3. Lancer le serveur
(.venv) $ uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

Ouvre ensuite http://localhost:8080

🐳 Docker (une seule commande)

Docker 
$ docker compose up
# Pour arrêter :
$ docker compose down
# Pour mettre à jour :
$ git pull && docker compose up --build

🔁 Arrêter le serveur

Tous OS 
# Dans le terminal où tourne uvicorn :
Ctrl + C

# Vérifier qu'aucun process traîne :
PS> netstat -ano | findstr :8080   # Win
$   lsof -i :8080                    # Mac/Linux
ℹ️

Premier lancement — patience requise

Au tout premier démarrage, PurpleForge télécharge automatiquement les données ATT&CK (~40 Mo) et les règles Sigma (~30 Mo). Cela prend 1 à 3 minutes. Les lancements suivants sont instantanés.

Tu verras dans le terminal des lignes comme Downloading ATT&CK data... — c'est normal, attends que Uvicorn running on http://0.0.0.0:8080 apparaisse.

🔑 Identifiants par défaut

ChampValeur par défautAction
Nom d'utilisateuradmin—
Mot de passeadmin⚠️ Changer immédiatement dans le panneau Compte

🔒 Mot de passe oublié

# Réinitialiser le mot de passe admin
(.venv) PS> python reset_password.py
# Suit les instructions à l'écran

Problème critique

Erreurs au démarrage

💥
Port déjà utilisé
error: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8080)
🔍 Cause : un autre programme (ou une ancienne instance de PurpleForge) occupe déjà le port 8080.

Solution A — changer le port :

(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8090 --reload
# Ouvrir http://localhost:8090

Solution B — trouver et tuer le processus qui bloque :

Windows 
PS> netstat -ano | findstr :8080
# Note le PID (dernière colonne) — ex: 15432
PS> taskkill /PID 15432 /F
macOSLinux 
$ lsof -i :8080
$ kill -9 <PID>
💥
Module introuvable
ModuleNotFoundError: No module named 'fastapi'
🔍 Cause : l'environnement virtuel n'est pas activé, ou les dépendances ne sont pas installées.

Vérifier que le venv est activé (tu dois voir (.venv) au début du prompt). Si non :

Windows 
PS> .\.venv\Scripts\Activate.ps1
macOSLinux 
$ source .venv/bin/activate

Puis réinstaller les dépendances :

(.venv) $ pip install -r requirements.txt
💥
Page blanche au démarrage
http://localhost:8080 — chargement infini ou page vide
🔍 Cause : les données ATT&CK sont en cours de téléchargement (premier lancement).

Regarde le terminal — tu dois voir des lignes de téléchargement :

INFO:     Downloading ATT&CK STIX data...
INFO:     Downloading Sigma rules...
INFO:     Uvicorn running on http://0.0.0.0:8080 (Press CTRL+C to quit)

Attends que la dernière ligne apparaisse, puis rafraîchis le navigateur. Si ça dure plus de 5 minutes, vérifie ta connexion internet et relance.

💥
Erreur de permission PowerShell (Windows)
Activate.ps1 cannot be loaded because running scripts is disabled on this system
🔍 Cause : la politique d'exécution PowerShell bloque les scripts.

Ouvre PowerShell en administrateur et tape :

PS> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
# Répondre "O" (Oui) à la confirmation

Ensuite relance PowerShell normalement et réactive le venv.

Environnement Python

Erreurs Python & environnement virtuel

🔧
Environnement virtuel corrompu / à recréer
Le venv est cassé ou les packages sont dans le mauvais état
🔍 Supprimer et recréer entièrement le venv :
Windows 
# Supprimer l'ancien venv
PS> Remove-Item -Recurse -Force .venv

# Recréer
PS> python -m venv .venv
PS> .\.venv\Scripts\Activate.ps1
(.venv) PS> pip install -r requirements.txt
macOSLinux 
$ rm -rf .venv
$ python3 -m venv .venv && source .venv/bin/activate
(.venv) $ pip install -r requirements.txt
🔧
Version Python incorrecte
SyntaxError: ... / This application requires Python 3.11+
🔍 Vérifier la version de Python utilisée :
PS> python --version
# Doit afficher Python 3.11.x ou 3.12.x

# Si "python" pointe sur Python 2 :
PS> python3 --version
PS> python3 -m venv .venv   # utiliser python3 explicitement

Si la version est < 3.11, télécharger Python 3.12 sur python.org/downloads.

📦 Mettre à jour les dépendances

# Vérifier les packages obsolètes
(.venv) PS> pip list --outdated

# Mettre à jour un package spécifique
(.venv) PS> pip install --upgrade fastapi

# Réinstaller toutes les dépendances proprement
(.venv) PS> pip install -r requirements.txt --upgrade

🔍 Vérifier que tout est installé

(.venv) PS> pip show fastapi uvicorn sqlmodel jinja2 itsdangerous mitreattack-python
# Chaque package doit afficher "Name:", "Version:", etc.
# Un message "WARNING: Package not found" indique un package manquant
⚠️

Toujours activer le venv avant de lancer

Si tu vois bash: uvicorn: command not found ou une erreur similaire, c'est presque toujours que le venv n'est pas activé. Le prompt doit afficher (.venv) en début de ligne.

Données externes

ATT&CK & Sigma — téléchargement

🌐
Téléchargement ATT&CK ou Sigma échoue
ConnectionError: Failed to download ATT&CK data / TimeoutError
🔍 Causes possibles : pas d'accès Internet, proxy d'entreprise, GitHub indisponible.

1. Vérifier la connexion Internet (ouvrir github.com dans le navigateur)

2. Si derrière un proxy — configurer pip :

(.venv) PS> pip install -r requirements.txt --proxy http://proxy.entreprise.com:3128

3. Forcer le re-téléchargement — supprimer le cache et relancer :

# Supprimer les données en cache (dossier data/)
PS> Remove-Item -Recurse data\*         # Windows
$   rm -rf data/*                        # Mac/Linux

# Garder le fichier marqueur vide
PS> New-Item data\.gitkeep -ItemType File -Force

# Relancer l'app — téléchargement repart de zéro
(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
🗂️
Matrice ATT&CK vide ou partielle
La matrice s'affiche sans techniques / certains onglets sont vides
🔍 Les données STIX ont été téléchargées partiellement (coupure réseau).

Supprimer le cache ATT&CK et relancer :

PS> Remove-Item data\attack_stix* -Force     # Windows
$   rm data/attack_stix*                     # Mac/Linux
(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
⚡
Bouton Sigma Rules — aucune règle affichée
Aucune règle trouvée pour cette technique
🔍 Causes possibles : règles Sigma non téléchargées, technique sans règle disponible.

Vérifier que les règles existent dans le cache :

PS> Get-ChildItem data\ | Where-Object Name -like "*sigma*"   # Windows
$   ls data/ | grep sigma                                    # Mac/Linux
# Doit afficher un dossier ou fichier sigma_rules

Si rien n'apparaît — supprimer data/sigma* et relancer l'app pour re-télécharger.

ℹ️ Technique sans règle

Certaines techniques très récentes ou trop génériques n'ont pas de règle Sigma dans SigmaHQ. C'est normal — cherche manuellement sur github.com/SigmaHQ/sigma.

Base de données

Base de données & migrations

🗄️
Erreur de table ou de colonne manquante
OperationalError: no such table: campaign / no such column: …
🔍 La base de données est en retard par rapport au schéma du code (après une mise à jour).

PurpleForge applique les migrations automatiquement au démarrage. Simplement relancer le serveur :

(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

Si l'erreur persiste, vérifier le log de démarrage pour la ligne Running migrations....

🗄️
Corrompre / réinitialiser la base
Database disk image is malformed / sqlite3.DatabaseError
🔍 Fichier SQLite corrompu (coupure de courant, disque plein…).

⚠️ Attention — perte de données

Supprimer la base efface toutes tes campagnes. Fais une sauvegarde d'abord si possible.

Sauvegarder d'abord (copier le fichier .db) :

PS> Copy-Item purpleforge.db purpleforge_backup.db

Puis supprimer la base pour repartir de zéro :

PS> Remove-Item purpleforge.db       # Windows
$   rm purpleforge.db              # Mac/Linux
# Relancer — la base sera recréée vide automatiquement
(.venv) PS> uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload

💾 Sauvegarder la base de données

La base de données est un simple fichier SQLite. Pour sauvegarder :

# Copie manuelle du fichier (app arrêtée de préférence)
PS> Copy-Item purpleforge.db "purpleforge_backup_$(Get-Date -Format 'yyyyMMdd').db"
💡

Backup hebdomadaire recommandé

Copie le fichier purpleforge.db dans un dossier cloud (OneDrive, Google Drive) chaque semaine. C'est ta seule protection contre une corruption.

Affichage

Interface & CSS — problèmes d'affichage

🎨
Page sans style — tout est en texte brut
La page s'affiche sans couleur ni mise en page
🔍 Cause la plus fréquente : le navigateur sert une version en cache de l'ancien CSS.

Solution rapide — vider le cache :

NavigateurRaccourci
Chrome / EdgeCtrl + Shift + R (Windows) / Cmd + Shift + R (Mac)
FirefoxCtrl + F5
SafariCmd + Option + R

Si ça ne suffit pas — ouvrir les DevTools (F12), onglet Network, cocher "Disable cache", puis rafraîchir.

En dernier recours — vider tout le cache du navigateur :

# Chrome : barre d'adresse
chrome://settings/clearBrowserData
# Cocher "Images and files" — Effacer les données
🎨
Favicon absent ou incorrect
L'onglet du navigateur montre le favicon par défaut au lieu de la mascotte
🔍 Fichier favicon.svg absent ou cache navigateur.

Vérifier que le fichier existe :

PS> Test-Path app\static\favicon.svg
# Doit afficher "True"

Si "False" — le fichier favicon.svg a été supprimé. Récupère-le depuis git :

PS> git checkout app/static/favicon.svg
🔄
Actions HTMX sans effet (boutons qui ne répondent pas)
Cliquer sur "+ Ajouter" ou changer un statut ne met pas à jour la page
🔍 JavaScript désactivé ou erreur JS dans la console.

Ouvrir les DevTools (F12) → onglet Console :

  • S'il y a des erreurs rouges → copie-les pour du support
  • Vérifier que HTMX est chargé : taper htmx.version dans la console — doit afficher "1.9.x"
  • Si HTMX absent → problème de réseau pour charger le CDN — vérifier la connexion

Tests automatisés

▶ Lancer les tests

# Lancer tous les tests (64 au total)
(.venv) PS> python -m pytest tests/ -v
64 passed in 0.75s ✓

# Tests rapides uniquement (4 checks de base)
(.venv) PS> python -m pytest tests/test_smoke.py -v

# Un test spécifique
(.venv) PS> python -m pytest tests/test_routes.py::TestCampaigns -v

# Avec rapport de couverture
(.venv) PS> python -m pytest tests/ -v --tb=short
🧪
Erreur httpx2 manquant
RuntimeError: The starlette.testclient module requires the httpx2 package
🔍 Package httpx2 non installé dans le venv.
(.venv) PS> pip install httpx2
# Relancer les tests
(.venv) PS> python -m pytest tests/ -v
🧪
Tests échouent — ImportError
ImportError: cannot import name 'app' from 'app.main'
🔍 Les tests doivent être lancés depuis la racine du projet, pas depuis un sous-dossier.
# Vérifier qu'on est à la racine :
PS> Get-Location
# Doit afficher : C:\...\purpleforge (pas C:\...\purpleforge\tests)

# Si on est dans le mauvais dossier :
PS> cd ..
(.venv) PS> python -m pytest tests/ -v
🧪
pytest introuvable
command not found: pytest / 'pytest' is not recognized
🔍 Utiliser python -m pytest au lieu de pytest directement.
# Toujours utiliser cette forme :
(.venv) PS> python -m pytest tests/ -v

# Installer pytest si vraiment absent :
(.venv) PS> pip install pytest

Conteneurisation

Docker

🐳 Commandes Docker essentielles

# Lancer (en arrière-plan)
$ docker compose up -d

# Voir les logs en temps réel
$ docker compose logs -f

# Arrêter
$ docker compose down

# Rebuild après une mise à jour du code
$ docker compose up --build

# Voir les conteneurs en cours
$ docker ps

# Entrer dans le conteneur (debug)
$ docker exec -it purpleforge_app bash
🐳
Docker Desktop non démarré
Cannot connect to the Docker daemon / docker: command not found
🔍 Docker Desktop n'est pas lancé.

Ouvrir Docker Desktop depuis le menu Démarrer / Applications, attendre que la baleine dans la barre des tâches soit stable (pas en mouvement), puis relancer docker compose up.

🐳
Image obsolète après une mise à jour
Les nouvelles fonctionnalités n'apparaissent pas dans la version Docker
🔍 Docker utilise encore l'ancienne image en cache.
# Forcer le rebuild complet
$ docker compose down
$ docker compose build --no-cache
$ docker compose up
🐳
Données perdues après docker compose down
Les campagnes disparaissent à chaque redémarrage Docker
🔍 Le volume Docker n'est pas correctement défini dans docker-compose.yml.

Vérifier que docker-compose.yml contient bien :

volumes:
    - ./purpleforge.db:/app/purpleforge.db
    - ./data:/app/data

Avec ce mapping, la base de données et les données ATT&CK survivent aux redémarrages.

Référence rapide

Checklist diagnostic — quand rien ne marche

À parcourir dans l'ordre quand tu ne sais pas d'où vient le problème :

  • ☐
    L'environnement virtuel est activé ? — Le prompt doit afficher (.venv)
  • ☐
    Le serveur tourne ? — uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload lancé sans erreur
  • ☐
    Tu ouvres le bon port ? — http://localhost:8080 (pas 8000, pas 3000)
  • ☐
    Le cache du navigateur est vidé ? — Ctrl+Shift+R
  • ☐
    Les dépendances sont installées ? — pip install -r requirements.txt
  • ☐
    La version Python est correcte ? — python --version → doit afficher 3.11+ ou 3.12+
  • ☐
    Les données ATT&CK sont téléchargées ? — Dossier data/ non vide
  • ☐
    Les 64 tests passent ? — python -m pytest tests/ -v
  • ☐
    Le code est à jour ? — git pull pour récupérer les dernières corrections
  • ☐
    Les logs du terminal indiquent quoi ? — Chercher une ligne ERROR ou CRITICAL

🏥 Commandes de diagnostic rapide

══ Environnement ══
(.venv) PS> python --version                         # doit afficher 3.11+
(.venv) PS> pip show fastapi uvicorn sqlmodel       # doit lister les versions

══ Réseau ══
PS> netstat -ano | findstr :8080               # port occupé ?
PS> Invoke-WebRequest http://localhost:8080    # app répond ?

══ Données ══
PS> Get-ChildItem data\                         # cache ATT&CK/Sigma présent ?
PS> Test-Path purpleforge.db                   # base de données existe ?

══ Tests ══
(.venv) PS> python -m pytest tests/ -v --tb=short  # état des tests

══ Logs du serveur ══
# Lancer et observer la console — chercher :
ERROR    ← problème critique, lire le message
WARNING  ← non bloquant mais à noter
INFO     ← fonctionnement normal
